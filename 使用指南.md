# MedRAG 医学文献助手 — 运行与使用指南

## 一、启动项目

```bash
# 进入项目目录
cd D:/dreamm/ceshi_rag

# 安装依赖（首次运行）
npm install

# 启动开发服务器
npm run dev
```

启动后终端显示：

```
VITE v5.4.x  ready in xxx ms
➜  Local:   http://localhost:3000/
```

浏览器打开 `http://localhost:3000`。

---

## 二、登录

| 用户名 | 密码 | 角色 | 说明 |
|---|---|---|---|
| `admin` | 任意 | 管理员 | 可访问聊天 + 知识库管理 + 用户管理 |
| `doctor_zhang` | 任意 | 普通用户 | 仅可访问聊天 |
| `researcher_li` | 任意 | 普通用户 | 仅可访问聊天（默认禁用） |

> 密码任意输入即可，mock 模式下不做校验。输入 `admin` 自动获得管理员权限。

**每个人的对话数据完全独立**——admin 创建的对话只有 admin 能看到，doctor_zhang 的对话只有 doctor_zhang 能看到，互不干扰。

## 三、进行对话

### 3.1 开始新对话

1. 登录后进入聊天页面
2. 点击左侧边栏的 **「发起新对话」** 按钮（虚线边框）
3. 在底部输入框输入医学问题，按 **Enter** 发送（Shift+Enter 换行）

### 3.2 推荐测试问题

项目内置了四个场景路由，根据你输入的关键词自动匹配：

| 场景 | 触发关键词 | 推荐输入 |
|---|---|---|
| 心血管风险评估 | `心血管`、`全科` | `基层全科医生如何进行心血管疾病风险评估？` |
| COPD 合并高血压 | `COPD`、`阻塞`、`高血压` | `COPD合并高血压患者的血压变异性与肺功能有什么关系？` |
| 基层卫生评价 | `评价`、`基层卫生` | `我国基层卫生服务评价指标体系的现状如何？` |
| 默认兜底 | 其他 | `介绍一下最新的医学研究进展` |

### 3.3 查看 AI 回复

每条 AI 回复包含：

- **RAG 检索过程**（可折叠）：意图识别 → 混合检索 → 融合重排 → 门神评估，每步可展开查看详情
- **Markdown 正文**：支持粗体、列表、表格、代码块等
- **文中引用标签**：蓝色 `[101]` `[102]` `[103]` 等，可直接点击
- **参考来源卡片**：引用文献的标题、摘要、来源期刊

---

## 四、查看文献 PDF（右侧面板）

### 4.1 从文中标签打开

1. 点击 AI 回复正文中的蓝色标签，如 `[101]`
2. 右侧滑出文献面板，自动加载对应 PDF
3. PDF 由 **vue-pdf-embed**（Mozilla pdf.js）渲染，支持：
   - 滚动浏览
   - 文本选中、复制
   - 缩放（浏览器右键）
   - 文本搜索（Ctrl+F）

### 4.2 从底部汇总按钮打开

1. 对话流末尾有一个 **「📚 参考文献 (N)」** 椭圆按钮
2. 点击后右侧面板以列表模式打开，显示当前对话涉及的所有文献
3. 点击列表中的任意文献卡片，切换到 PDF 详情视图

### 4.3 面板操作

| 操作 | 方式 |
|---|---|
| 返回列表 | 点击 PDF 视图顶部的 **「← 返回文献列表」** |
| 关闭面板 | 点击面板右上角 **✕** |

---

## 五、管理对话历史

### 5.1 切换对话

点击左侧边栏中的历史对话条目即可切换。当前对话高亮显示。

### 5.2 删除对话

鼠标悬停在对话条目上，点击右侧出现的 **🗑️ 删除** 图标：

- 删除**当前对话**：自动回到空状态页面
- 删除**历史对话**：不影响当前对话

### 5.3 持久化

- 所有对话和消息**自动保存到浏览器 localStorage**
- 刷新页面不会丢失
- 每个用户的数据独立存储，切换账号不会串数据

---

## 六、管理后台（仅 admin）

点击左侧边栏底部的管理入口：

### 知识库管理（`/admin/knowledge`）

- 查看三篇已上传的医学文献（含文件大小、切片数、状态）
- 上传新文档（mock 模式下模拟成功）
- 删除文档

### 用户管理（`/admin/users`）

- 查看系统用户列表（用户名、角色、最后登录时间）
- 启用/禁用用户

---

## 七、侧边栏操作

| 操作 | 方式 |
|---|---|
| 收起侧边栏 | 点击顶部 header 的 **◀** 按钮 |
| 展开侧边栏 | 点击顶部 header 的 **▶** 按钮 |
| 关闭侧边栏（键盘） | 按 **ESC** 键 |
| 移动端关闭 | 点击侧边栏外的半透明遮罩 |

---

## 八、切回真实后端

当后端服务可用时：

1. 打开 `src/mock/index.ts`，将 `USE_MOCK` 改为 `false`
2. 打开 `vite.config.ts`，取消 `/api` proxy 的注释，将 `target` 改为后端地址
3. 重启 `npm run dev`
